whiteprocess
Whitelist Threads Filter for UNIX Server

Antivirus and Threads Controller for High Security Servers







Introduction

whiteprocess is a simple software programmed in Python 2.7 and compiled in ELF for UNIX systems which is used to filter threads in whitelist.
Its purpose is to optimize the work that antivirus offers, performing a filtering of all process/threads that the user did not declare to be "allowed" to run.
whiteprocess replaces antivirus on server.

Its objectives are:
  • Blocking malwares and unwanted executables (executables and scripts)
  • Blocking buffer overflow attacks
  • Regularly check the running threads
  • Offer a user-friendly system

Main Features

  • Filter Threads by Executables
    • Given a list of executables, it kills thread initiated by executables not belonging to the list.

  • Filter New Spawned Threads
    • Given a list of executables, all threads created after running whiteprocess will been killed (useful against many buffer overflow attacks).

  • Filter Threads by Arguments
    • Filter threads from an arguments whitelist instead of executables whitelist.

  • Remote Control
    • Possibility to control whiteprocess on multiple servers with a single command.

  • Modular
    • whiteprocess is designed to be modular.
    It consists of several modules that can be exchanged flexibly.

  • Simple Structure
    • whiteprocess has a simple structure that will be improved and optimized over time. This simplicity makes future updates and customizations simple.
    No external dependencies required.

  • High Security
    • This software offers a high level of security, even in its implementation.

  • Open Source
    • whiteprocess is entirely open source and can be modified and customized to suit your needs.
